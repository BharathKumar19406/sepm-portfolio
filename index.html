Introduction to Software Engineering

Structuring defense in depth.

  • Evolving Threats: Security threats change constantly, so engineering is about preparing for new attack vectors.
  • Layered Architecture: Designed RASP with a clear separation between the core logic and the presentation layer.
  • Modular Design: Separated monitoring, logging, and mitigation workflows so the system can scale without muddying components.
  • Forensic Cleanliness: Ensured the event database records actionable telemetry without UI clutter.

Process Models & Management

Choosing the right path for delivery.

  • Agile Methodology: Allowed the team to respond to security testing feedback rapidly, adapting rules as new vulnerabilities were discovered.
  • Iterative Hardening: Started with basic request logging, then incrementally added complex features like rate limiting (10 req/sec) and payload inspection.
  • Sprint Reviews: Used short cycles to fix rate-limit violations and ensure a `403 Forbidden` response for robust mitigation.
  • Continuous Improvement: Adopted frequent security audits to keep the RASP dashboard focused and forensic.

Requirements Engineering

Turning security needs into actionable tasks.

Functional Requirements

  • Real-time inspection of HTTP requests for XSS, SQLi, and SSRF.
  • Enforce a global site-wide rate limit to block fuzzers like `dirb`.
  • Provide a forensic dashboard for SOC analysts to export logs.

Non-Functional Requirements

  • Performance: Minimal latency overhead (<10ms) per request.
  • Security: Prevent information disclosure on blocks (return generic 403).
  • Maintainability: Clean, decoupled Python modules for easy rule updates.

Architecture & UI/UX Design

Mapping the structure and experience.

Software Architecture

Organized the system so each part has a clear, singular responsibility:

  • Monitoring: Captures raw telemetry efficiently.
  • Analysis: Flags anomalies and potential threats dynamically.
  • Extraction: Parses payloads for deeper inspection.
  • Dashboard: Streamlit-based UI for exporting and viewing forensic data.

UI/UX Design

Designed for SOC analysts who need speed and clarity:

  • Removed all unnecessary visual clutter ("Attack Analysis" graphics).
  • Implemented a strictly forensic, high-performance monitoring interface.
  • Dark mode aesthetics to reduce eye strain during prolonged monitoring sessions.

Software Construction

Writing code with discipline.

  • Clean Code: Organized Python code into clear, independent modules.
  • Configuration Management: Centrally managed rate limits, thresholds, and regex patterns.
  • Refactoring: Streamlined the dashboard by removing redundant UI elements, focusing solely on actionable intelligence.
  • Version Control: Maintained strict commit hygiene while pushing critical security updates to the GitHub repository.

Testing & Risk Management

Validating defenses and controlling uncertainty.

Testing Strategies

  • Adversarial Testing: Simulated attacks using tools like `ffuf` and `dirb` to verify rate limiting.
  • Payload Validation: Tested rule sets against known malicious inputs (Path Traversal, SQLi) to ensure accurate blocking.
  • Integration Testing: Verified that blocked requests properly registered in the database with accurate source/target IP tracking.

Risk Management (RMMM)

  • Risk: High false-positive rate blocking legitimate traffic.
  • Mitigation: Fine-tuned the analysis engine with baseline profiles.
  • Risk: Information disclosure via verbose error messages.
  • Mitigation: Hardcoded standard `403 Forbidden` responses for all security violations.

Conclusion & Future Scope

Looking ahead.

Working on the RASP project demonstrated that Software Engineering Project Management is crucial for building reliable security tools. Balancing performance with robust defense requires careful planning, iterative testing, and a disciplined approach to architecture.

Future Scope (Maintenance)

  • Adaptive Maintenance: Updating rule sets to counter zero-day exploits.
  • Perfective Maintenance: Implementing distributed rate-limiting across multiple instances via Redis.
  • Preventive Maintenance: Automating the CI/CD pipeline for regular dependency vulnerability scanning.
